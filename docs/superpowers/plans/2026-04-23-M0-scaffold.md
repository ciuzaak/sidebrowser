# M0：工程脚手架 — 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 搭建 sidebrowser 的工程脚手架：Electron + electron-vite + React 19 + TypeScript strict + Tailwind v4 + Vitest + Playwright for Electron。完成后 `pnpm dev` 能启动一个跑通 IPC ping 的 hello-world 窗口，`pnpm test` 和 `pnpm test:e2e` 都通过。

**Architecture:** main/preload/renderer 三进程隔离；Renderer 走 React 19 + Vite HMR；IPC 走 `contextBridge` 暴露的类型化 API；测试分单元（Vitest）和 E2E（Playwright `_electron`）两层。

**Tech Stack:** electron, electron-vite, React 19, TypeScript 5.7+ strict, Tailwind CSS v4（@tailwindcss/vite 插件，零配置）, Vitest, @playwright/test, pnpm, ESLint, Prettier.

**Spec 参考：** `docs/superpowers/specs/2026-04-23-sidebrowser-design.md` §3 技术栈 / §4.3 目录结构。

**注意（目录结构偏差）：** Spec §4.3 写的是 `src/renderer/main.tsx`，实际使用 electron-vite 默认约定会是 `src/renderer/index.html` + `src/renderer/src/main.tsx`。本计划沿用默认约定；后续里程碑按此实际路径推进，spec 的示意结构理解为"大致位置"即可。

## Task 5: 创建 main 进程入口（最小 BrowserWindow + ping handler）

**Files:**
- Create: `src/main/index.ts`

- [ ] **Step 1: 创建 `src/main/index.ts`**

```ts
import { app, BrowserWindow, ipcMain } from 'electron';
import { fileURLToPath } from 'node:url';
import { dirname, join } from 'node:path';
import { IpcChannels } from '@shared/ipc-contract';

const __dirname = dirname(fileURLToPath(import.meta.url));

function createWindow(): BrowserWindow {
  const win = new BrowserWindow({
    width: 393,
    height: 852,
    title: 'sidebrowser',
    webPreferences: {
      preload: join(__dirname, '../preload/index.mjs'),
      sandbox: true,
      contextIsolation: true,
      nodeIntegration: false,
    },
  });

  if (process.env['ELECTRON_RENDERER_URL']) {
    void win.loadURL(process.env['ELECTRON_RENDERER_URL']);
  } else {
    void win.loadFile(join(__dirname, '../renderer/index.html'));
  }

  return win;
}

function registerIpcHandlers(): void {
  ipcMain.handle(IpcChannels.appPing, (_event, payload: { message: string }) => {
    return {
      reply: `pong: ${payload.message}`,
      timestamp: Date.now(),
    };
  });
}

app.whenReady().then(() => {
  registerIpcHandlers();
  createWindow();

  app.on('activate', () => {
    if (BrowserWindow.getAllWindows().length === 0) {
      createWindow();
    }
  });
});

app.on('window-all-closed', () => {
  if (process.platform !== 'darwin') {
    app.quit();
  }
});
```

```tsx
      </code>

}
```

## Task 11: 配置 ESLint（flat config v9+） + Prettier

**Files:**
- Create: `eslint.config.js`
- Create: `.prettierrc.json`
- Create: `.prettierignore`
- Modify: `package.json`（把 `lint` script 指向 flat config）

ESLint v9+ 默认 flat config（`eslint.config.js`），`.eslintrc.*` 是 legacy。React plugin 的 flat config 导出从 eslint-plugin-react v7.35+ 起可用。

- [ ] **Step 1: 安装 flat config 需要的额外包**

```bash
pnpm add -D @eslint/js@latest globals@latest
```

（`@eslint/js` 提供官方 recommended 规则，`globals` 提供环境全局变量名。）

- [ ] **Step 2: 创建 `eslint.config.js`**

```js
import js from '@eslint/js';
import tseslint from '@typescript-eslint/eslint-plugin';
import tsparser from '@typescript-eslint/parser';
import react from 'eslint-plugin-react';
import reactHooks from 'eslint-plugin-react-hooks';
import globals from 'globals';

export default [
  {
    ignores: [
      'node_modules/**',
      'out/**',
      'dist/**',
      'coverage/**',
      'playwright-report/**',
      'test-results/**',
    ],
  },
  js.configs.recommended,
  {
    files: ['**/*.{ts,tsx}'],
    languageOptions: {
      parser: tsparser,
      parserOptions: {
        ecmaVersion: 'latest',
        sourceType: 'module',
        ecmaFeatures: { jsx: true },
      },
      globals: {
        ...globals.browser,
        ...globals.node,
      },
    },
    plugins: {
      '@typescript-eslint': tseslint,
      react,
      'react-hooks': reactHooks,
    },
    settings: { react: { version: 'detect' } },
    rules: {
      ...tseslint.configs.recommended.rules,
      ...react.configs.recommended.rules,
      ...reactHooks.configs.recommended.rules,
      'react/react-in-jsx-scope': 'off',
      'react/prop-types': 'off',
      '@typescript-eslint/no-unused-vars': ['warn', { argsIgnorePattern: '^_' }],
      '@typescript-eslint/consistent-type-imports': 'error',
      'no-undef': 'off', // TS 已经处理了；ESLint 的实现对 TS 类型不友好
    },
  },
];
```

- [ ] **Step 3: 创建 `.prettierrc.json`**

```json
{
  "semi": true,
  "singleQuote": true,
  "trailingComma": "all",
  "printWidth": 100,
  "tabWidth": 2,
  "endOfLine": "lf"
}
```

- [ ] **Step 4: 创建 `.prettierignore`**

```
node_modules
out
dist
coverage
pnpm-lock.yaml
playwright-report
test-results
```

- [ ] **Step 5: 修改 `package.json` 里的 `lint` 脚本去掉旧的 `--ext` 参数**

flat config 不用 `--ext`（直接在 config 里用 `files` 决定）。把 package.json 中的：

```json
"lint": "eslint . --ext .ts,.tsx",
```

改成：

```json
"lint": "eslint .",
```

- [ ] **Step 6: 跑 Lint 看现状**

```bash
pnpm lint
```

Expected: 0 errors。可能有 0 个或少量 warning。若有 warning（比如未使用的 import）修掉再进下一步。

- [ ] **Step 7: 跑 Format 统一风格**

```bash
pnpm format
```

Expected: 修改若干文件的格式。`git diff` 确认改动都是格式性的（空格、引号、换行）。

- [ ] **Step 8: Commit**

```bash
git add eslint.config.js .prettierrc.json .prettierignore package.json
git add -u
```
