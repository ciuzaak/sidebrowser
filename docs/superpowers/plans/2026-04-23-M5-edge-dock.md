# M5：贴边自动缩出 — 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 窗口贴左/右屏幕边（距边 ≤ `edgeThresholdPx`）时，鼠标离开 → 200ms 动画把窗口 X 推出屏，只留 `triggerStripPx` 在屏幕上；鼠标再回到触发条上 → 反向动画恢复。M4 的 DOCKED_NONE dim 路径继续走，新增 DOCKED_*/HIDDEN_* 状态链路。Mid-animation cancel（隐藏中途鼠标回来）反向走 reveal。状态广播给 renderer 让 TopBar 淡出。

**Architecture:**
- 纯函数层：`computeDockedSide(bounds, workArea, threshold) → 'left' | 'right' | null`；`interpolateX(from, to, progress) → number`（ease-out-cubic）；`reduce(state, event, config) → { nextState, effects[] }`。3 个文件，100% 单测。
- `src/main/edge-dock-reducer.ts`：纯 reducer。state/event/effect 三种 union。spec §5.1 全量状态转移编码成 switch。
- `src/main/edge-dock.ts`：effects executor。吃 reducer 输出，执行 `setInterval(16ms)` 插值动画、调 DimController、广播 window:state。持有 `currentX` 跟踪动画进度（reducer 不碰真实 bounds）。
- `src/shared/ipc-contract.ts`：加 `window:state` 通道 + `WindowState` 类型。
- `main/index.ts`：把 M4 的直连（`watcher.onLeave → dim.apply`）替换为 `edgeDock.dispatch(MOUSE_LEAVE)`。追加 `win.on('moved')` / `screen.on('display-metrics-changed')` 两个事件源。`viewManager.onSnapshot → dim.retarget` 保留（M4 行为，和 EdgeDock 无关）。
- Renderer：`preload` 暴露 `onWindowState(cb)`；TopBar 订阅后按 `hidden` 切 Tailwind fade class。不动 TabDrawer。
- E2E 动画不 mock：保留 200ms 真实动画，`expect.poll` 等 `getWindowBounds().x` 到位（总体时间还在 5s poll 预算内）。用 M4 已有的 `SIDEBROWSER_E2E=1` 测试钩机制，追加 `emitWindowMoved / emitDisplayChanged / getEdgeDockState / getWindowBounds` 几个钩子。

**Tech stack delta vs M4:** 无新依赖。

**Spec references:** §2（"鼠标离开"定义复用 M4）、§5.1（状态机 + 事件源 + reducer/executor 分层 + Windows `setBounds` 动画方式）、§6（`window:state` 通道已在 spec §6 表里声明 M→R event —— M5 落地）、§7（`window.edgeThresholdPx`、`window.width`、`edgeDock.{enabled, animationMs, triggerStripPx}`）、§8.2（DOCKED_RIGHT → HIDING 全链路示例）、§10（`HIDING/REVEALING` 期间忽略新事件的"动画中不丢事件"原则；显示器热插拔 → snap 回主屏中心）、§12（E2E 策略：reducer 100% 单测；E2E 可调 `animationMs=0` 但本计划选择保留真实动画 + poll）、§13（M5 里程碑）。

**M5 特定 guardrails（M4 留下的 + 新发现）：**
- `CursorWatcher.onLeave` 已自带 `delayMs` 防抖；reducer 收到 `MOUSE_LEAVE` 时就是"人已离开且超过延迟"。reducer 不再加 debounce。
- `CursorWatcher.start()` 在 E2E 模式仍然被 gate（M4 `fix(main)` 的 fix），测试钩用 `fireLeaveNow/fireEnterNow` 注入。M5 的 `emitWindowMoved` / `emitDisplayChanged` 走同一套路。
- Electron `win.on('moved')` 在 Windows 只在 drag-drop 后 fire 一次；reducer `WINDOW_MOVED` 不需要额外 debounce。`setBounds` 自身不触发 `moved`，动画内不会自我触发。
- 动画期间 (HIDING/REVEALING) 忽略 `WINDOW_MOVED`（用户不会拖隐藏窗口；setBounds 自身不 fire）。`MOUSE_ENTER` 在 HIDING 期间是合法取消信号。
- `setBounds(bounds, animate)` 的 `animate` 只在 macOS 生效；Windows 走 `setInterval(16ms)` 插值。不传 `animate` 参数。
- `screen.getDisplayMatching(bounds)` 给当前窗口所在 display；如果 bounds 完全离屏（DISPLAY_CHANGED 导致），`getDisplayNearestPoint(bounds 中心)` 兜底。
- `webContents.insertCSS` 的 Promise<key> 已由 M4 DimController 处理；M5 只 reuse，不动。
- E2E 不 mock `screen`：用测试钩 `emitDisplayChanged` 代替真实事件源注入（类似 M4 fireLeaveNow）。
- 每个 E2E spec 用 `mkdtempSync(join(tmpdir(), 'sidebrowser-m5-XXX-'))` 做 userData 隔离；`playwright.config.ts` 不动。
- `window:state` 广播的 `dimmed` 字段和 `DimController.isActive` 同步；`hidden` 对应 `HIDDEN_*`；`docked` 对应 `'left' | 'right' | null`。renderer 不直接碰 DimController，只读广播。

**M5 Definition of Done:**
- 窗口贴右边（距 display.x+width ≤ 8px）时，鼠标离开窗口矩形 → 100ms 后 dim + 200ms 动画推出屏，只剩 3px 在右边。
- 鼠标 hover 到那 3px 触发条 → 反向动画把窗口拖回，dim 清除。
- 贴左边同理。
- 不贴边（DOCKED_NONE）的鼠标离开：M4 行为——只 dim，不动窗口。
- 动画中鼠标回来（HIDING 期间） → 取消隐藏动画，走 REVEALING 回 docked 位置，dim 清除。
- WINDOW_MOVED 拖离边 → 状态回 DOCKED_NONE，不动窗口。
- 显示器 unplug / 分辨率变 → 若 bounds 仍在某 display 内：状态回 DOCKED_NONE；若完全离屏：snap 回主屏 workArea 中心。
- Renderer 订阅 `window:state`，TopBar 在 `hidden=true` 时加淡出 class（opacity 0.3，transition 200ms）；`hidden=false` 恢复。
- 单测覆盖：`computeDockedSide`（≥5 case）、`interpolateX`（≥4 case）、`edge-dock-reducer`（每条转移 ≥1 case，共 ≥12）、`edge-dock` executor（mock 计时器 + spy deps，≥6 case）。
- E2E ≥1 新 spec 覆盖：dock detection、hide animation、reveal animation、mid-hide cancel、DISPLAY_CHANGED snap。
- M4 M3 功能不倒退（dim DOCKED_NONE、tab、UA、favicon）。

## Task 2: computeDockedSide + interpolateX 纯函数 + 单测

**Files:** Create `src/main/edge-geometry.ts`（两个函数 co-located）、`tests/unit/edge-geometry.test.ts`

```ts
export function computeDockedSide(
  bounds: { x: number; width: number },
  workArea: { x: number; width: number },
  edgeThresholdPx: number,
): 'left' | 'right' | null {
  if (Math.abs(bounds.x - workArea.x) <= edgeThresholdPx) return 'left';
  if (Math.abs((bounds.x + bounds.width) - (workArea.x + workArea.width)) <= edgeThresholdPx) return 'right';
  return null;
}

export function interpolateX(from: number, to: number, progress: number): number {
  const t = Math.max(0, Math.min(1, progress));
  const eased = 1 - Math.pow(1 - t, 3); // ease-out-cubic
  return from + (to - from) * eased;
}
```

**TDD tests:**
- `computeDockedSide`: left edge aligned → 'left'; right edge aligned → 'right'; within threshold (exact = threshold) → 左/右；within threshold + 1 → null；both edges near but left wins（左先判）；负 workArea.x（多显示器）→ 逻辑正确。≥5 case。
- `interpolateX`: progress=0 → from；progress=1 → to；progress=0.5 → easing 使值 > 线性中点（ease-out 前段快）；progress clamped <0 / >1；from>to（反向）→ 依然单调。≥4 case。

## Task 3: edge-dock-reducer.ts — 纯状态机 + 全量单测

**Files:** Create `src/main/edge-dock-reducer.ts`、`tests/unit/edge-dock-reducer.test.ts`

最重的一个 Task。reducer 必须 100% 覆盖 spec §5.1 状态转移表。

```ts
export type EdgeDockState =
  | { kind: 'DOCKED_NONE' }
  | { kind: 'DOCKED_LEFT' | 'DOCKED_RIGHT' }
  | { kind: 'HIDING' | 'REVEALING'; side: 'left' | 'right' }
  | { kind: 'HIDDEN_LEFT' | 'HIDDEN_RIGHT' };

export type EdgeDockEvent =
  | { type: 'MOUSE_LEAVE' }
  | { type: 'MOUSE_ENTER' }
  | { type: 'WINDOW_MOVED'; bounds: Rect; workArea: Rect }
  | { type: 'DISPLAY_CHANGED'; bounds: Rect; workArea: Rect | null }
  | { type: 'ANIM_DONE' };

export type EdgeDockEffect =
  | { type: 'APPLY_DIM' }
  | { type: 'CLEAR_DIM' }
  | { type: 'ANIM_HIDE'; side: 'left' | 'right'; targetX: number; ms: number }
  | { type: 'ANIM_REVEAL'; side: 'left' | 'right'; targetX: number; ms: number }
  | { type: 'ANIM_CANCEL' }
  | { type: 'SNAP_TO_CENTER'; workArea: Rect; windowWidth: number }
  | { type: 'BROADCAST_STATE'; docked: 'left' | 'right' | null; hidden: boolean; dimmed: boolean };

export interface EdgeDockConfig {
  edgeThresholdPx: number;
  animationMs: number;
  triggerStripPx: number;
  windowWidth: number;
  enabled: boolean;
}

export function reduce(
  state: EdgeDockState,
  event: EdgeDockEvent,
  cfg: EdgeDockConfig,
): { nextState: EdgeDockState; effects: EdgeDockEffect[] };
```

**核心转移表（覆盖 spec §5.1 + §10 动画中事件处理）：**

| state | event | nextState | effects |
|---|---|---|---|
| DOCKED_NONE | MOUSE_LEAVE | DOCKED_NONE | APPLY_DIM, BROADCAST(docked=null, hidden=false, dimmed=true) |
| DOCKED_NONE | MOUSE_ENTER | DOCKED_NONE | CLEAR_DIM, BROADCAST(docked=null, hidden=false, dimmed=false) |
| DOCKED_LEFT/RIGHT | MOUSE_LEAVE | HIDING side=同 | APPLY_DIM, ANIM_HIDE(targetX 见下), BROADCAST(docked=side, hidden=false, dimmed=true) |
| DOCKED_LEFT/RIGHT | MOUSE_ENTER | DOCKED_* | CLEAR_DIM, BROADCAST(docked=side, hidden=false, dimmed=false) |
| HIDING | ANIM_DONE | HIDDEN_{SIDE} | BROADCAST(docked=side, hidden=true, dimmed=true) |
| HIDING | MOUSE_ENTER | REVEALING side=同 | ANIM_CANCEL, CLEAR_DIM, ANIM_REVEAL(targetX=dockedX), BROADCAST(docked=side, hidden=false, dimmed=false) |
| HIDDEN_LEFT/RIGHT | MOUSE_ENTER | REVEALING side=同 | CLEAR_DIM, ANIM_REVEAL(targetX=dockedX), BROADCAST(docked=side, hidden=false, dimmed=false) |
| REVEALING | ANIM_DONE | DOCKED_{SIDE} | （已在进入 REVEALING 时广播；此处无 effect，或重播 BROADCAST 幂等 OK） |
| REVEALING | MOUSE_LEAVE | REVEALING | 忽略（动画中不接受新事件；spec §10） |
| DOCKED_* | WINDOW_MOVED（off edge） | DOCKED_NONE | BROADCAST(docked=null, hidden=false, dimmed=现值) |
| DOCKED_NONE | WINDOW_MOVED（on edge） | DOCKED_{side} | BROADCAST(docked=side, hidden=false, dimmed=现值) |
| DOCKED_* | WINDOW_MOVED（still same side） | 同 | 无 effect（幂等） |
| HIDING/REVEALING/HIDDEN_* | WINDOW_MOVED | 同 | 忽略（setBounds 自身不 fire moved，用户侧极少碰到；守护性忽略） |
| any | DISPLAY_CHANGED（bounds 在某 workArea 内） | DOCKED_NONE | CLEAR_DIM（如果 dimmed）, BROADCAST(docked=null, hidden=false, dimmed=false) |
| any | DISPLAY_CHANGED（bounds 完全离屏） | DOCKED_NONE | CLEAR_DIM, SNAP_TO_CENTER, BROADCAST(docked=null, hidden=false, dimmed=false) |

**targetX 计算：**
- `side='left'` HIDE：`workArea.x - windowWidth + triggerStripPx`
- `side='right'` HIDE：`workArea.x + workArea.width - triggerStripPx`
- `side='left'` REVEAL：`workArea.x`
- `side='right'` REVEAL：`workArea.x + workArea.width - windowWidth`

reducer 不直接持有 workArea——`MOUSE_LEAVE` 事件 payload 里传 `bounds + workArea` 吗？更干净的做法：reducer 内部存"最后一次知道的 workArea"。每次 `WINDOW_MOVED` / `DISPLAY_CHANGED` 更新；`MOUSE_LEAVE` 时用缓存的 workArea + cfg.windowWidth 算 targetX。

**状态形状补充：**

```ts
export type EdgeDockState =
  | { kind: 'DOCKED_NONE'; workArea: Rect | null }
  | { kind: 'DOCKED_LEFT' | 'DOCKED_RIGHT'; workArea: Rect }
  | ...
```

每个状态带它需要的 workArea。初始值：`{ kind: 'DOCKED_NONE', workArea: null }`，启动时 bootstrap 发一个 WINDOW_MOVED 种下 workArea。

**`cfg.enabled = false` 分支：** reducer 对所有事件回 `{ nextState: state, effects: [] }`（EdgeDock 完全禁用）。M5 Definition of Done 不强制这个分支跑通，但 reducer 该守护。

**TDD tests (≥ 12)：** 每条转移 ≥1 case；加：enabled=false 全部 no-op；DOCKED_RIGHT 同 WINDOW_MOVED 幂等；WINDOW_MOVED 从 DOCKED_LEFT 到 DOCKED_RIGHT 的直接切换；HIDING + WINDOW_MOVED 忽略。

## Task 5: edge-dock.ts — effects executor + 单测

**Files:** Create `src/main/edge-dock.ts`、`tests/unit/edge-dock.test.ts`

依赖注入让单测 Node-only：

```ts
export interface EdgeDockDeps {
  setWindowX: (x: number) => void;          // win.setBounds({ x, ... })
  getWindowBounds: () => Rect;
  applyDim: () => void;
  clearDim: () => void;
  broadcastState: (s: WindowState) => void;
  now: () => number;                        // Date.now() in prod, fake in tests
  setInterval: (cb: () => void, ms: number) => IntervalHandle;
  clearInterval: (h: IntervalHandle) => void;
  config: EdgeDockConfig;
}

export class EdgeDock {
  private state: EdgeDockState = { kind: 'DOCKED_NONE', workArea: null };
  private anim: { handle: IntervalHandle; fromX: number; toX: number; startedAt: number; ms: number } | null = null;

  constructor(private readonly deps: EdgeDockDeps) {}

  dispatch(event: EdgeDockEvent): void {
    const { nextState, effects } = reduce(this.state, event, this.deps.config);
    this.state = nextState;
    for (const fx of effects) this.runEffect(fx);
  }

  getState(): EdgeDockState { return this.state; }

  private runEffect(fx: EdgeDockEffect): void { /* switch on fx.type */ }

  private startAnim(side, targetX, ms): void {
    this.cancelAnim();
    if (ms === 0) {
      this.deps.setWindowX(targetX);
      this.dispatch({ type: 'ANIM_DONE' });
      return;
    }
    const fromX = this.deps.getWindowBounds().x;
    const startedAt = this.deps.now();
    const handle = this.deps.setInterval(() => {
      const t = Math.min(1, (this.deps.now() - startedAt) / ms);
      this.deps.setWindowX(interpolateX(fromX, targetX, t));
      if (t >= 1) {
        this.cancelAnim();
        this.dispatch({ type: 'ANIM_DONE' });
      }
    }, 16);
    this.anim = { handle, fromX, toX: targetX, startedAt, ms };
  }

  private cancelAnim(): void { if (this.anim) { this.deps.clearInterval(this.anim.handle); this.anim = null; } }
}
```

**注意：**
- executor 把 `MOUSE_LEAVE` 的 targetX 计算下放给 reducer（reducer effect 已带 `targetX`）。executor 只执行 `setWindowX(x)`。
- reducer 不知道"当前真实 X"；executor 从 `getWindowBounds()` 读。这是两层分工关键点。
- `ANIM_CANCEL` effect → `cancelAnim()`；随后的 `ANIM_REVEAL` effect → `startAnim` 用当前真实 X 作 fromX。
- `SNAP_TO_CENTER` effect：`setWindowX(workArea.x + (workArea.width - windowWidth) / 2)`。

**TDD tests (≥ 6)：**
- ANIM_HIDE with ms=0 → setWindowX 立即 toX + dispatch ANIM_DONE（断言 state.kind === 'HIDDEN_*'）。
- ANIM_HIDE with ms=200 + fake timers 推进 200ms → setWindowX 多次调用，最终 x === toX；ANIM_DONE 触发。
- ANIM_HIDE 中途（t=0.5）dispatch MOUSE_ENTER → cancelAnim → ANIM_REVEAL 从当前 x 开始（断言 startAnim.fromX === 中途值）。
- APPLY_DIM → applyDim() 被调用；CLEAR_DIM → clearDim()。
- BROADCAST_STATE effect → broadcastState 被调用、payload 和 effect 字段一一对应。
- SNAP_TO_CENTER → setWindowX(正确居中值)。

**Mock helper 模式：** `mk()` 工厂，返回 `{ deps, getSetBoundsCalls, getBroadcastCalls, ... }`。`setInterval/clearInterval/now` 用 `vi.useFakeTimers()` + 让 `now: () => Date.now()`（`vi.setSystemTime` 控制）。

## Task 8: E2E — dock / hide / reveal / cancel / display-change

**Files:** Create `tests/e2e/edge-dock.spec.ts`

**Launch pattern:** 复用 M4 的 `SIDEBROWSER_E2E=1` env，追加的测试钩 `emitWindowMoved / emitDisplayChanged / getEdgeDockState / getWindowBounds / setWindowBounds` 在 Task 6 里已挂好。

**测试流（单 test，一次 launch）：**

1. launch → getChromeWindow → waitForAddressBarReady → navigate `/plain`。
2. **Dock detection：** `setWindowBounds({ x: display.workArea.x, y: ..., width: 393, height: 852 })` 把窗口贴左边 → `emitWindowMoved` → poll `getEdgeDockState()` → 断言 `state.kind === 'DOCKED_LEFT'`。
3. **Hide animation：** `fireLeaveNow` → poll `getWindowBounds().x` → 断言最终 `x === workArea.x - 393 + 3`（左贴 HIDDEN 位置）且 `getEdgeDockState().kind === 'HIDDEN_LEFT'`。`timeout: 5000` 足够覆盖 200ms 动画 + 缓冲。
